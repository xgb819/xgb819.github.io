# 前言

最近家里的笔记本 C 盘爆满了，加上之前一直诟病 Win11 登录账户后锁屏无法进入系统的 bug，最近看公司电脑装新版 Win11 已经修复了，就想着趁这个机会，把电脑彻底清空重置，换成纯净的新版 Win11。本来想着只是简单重装，没想到接连踩坑，折腾了好久才彻底搞定。

开始像之前给公司电脑重装系统一样，去 [微软官网](https://www.microsoft.com/zh-cn/software-download/windows11) 下载官方安装助手，按照步骤运行，到关机重启执行安装的关键步骤，直接中断，提示安装失败，未执行任何更改，直接回滚到原来的系统，电脑文件、系统完全没变动。重置此电脑也出现同样的问题，尝试修复 Windows RE、使用 SFC 修复系统文件、从恢复环境中进行启动修复（没有相关选项）等都不起作用。

后来，改用 U 盘重装，成功进入安装向导界面了，但是识别不到固态硬盘。最后通过另一个 U 盘手动加载 Intel RST 硬盘驱动，成功识别到硬盘并安装系统。

# 前期准备

在 Win11 安装过程中，手动加载 Intel RST 硬盘驱动，让安装程序识别到 NVMe 硬盘，全程需要两个U盘配合。

1. U 盘 1（系统盘）：容量 ≥ 8GB，用微软官方 MediaCreationTool 工具，制作 Win11 安装启动盘；

2. U 盘 2（驱动盘）：容量 ≥ 4GB，空 U 盘即可，提前下载好 Intel RST VMD 驱动程序（去对应电脑品牌官方支持 / 驱动下载页找到对应机型的 Intel RST 驱动包），解压后把整个驱动文件夹放进 U 盘根目录，方便后续查找。

# 安装步骤

1. 关机插盘：电脑关机，同时插上系统 U 盘和驱动 U 盘，两个 U 盘都插在电脑的 USB 接口上，建议插后置接口，避免拓展坞识别不稳；

2. 进入 BIOS 调启动项：开机瞬间，根据自己电脑品牌反复按对应快捷键，进入 BIOS 设置界面，找到 Boot（启动）选项，把 Win11 系统安装 U 盘设为第一启动项；

3. 找到 Secure Boot（安全启动），改成 Disabled（关闭），保存设置并重启（一般按 <kbd>F10</kbd> 保存退出）；

4. 电脑重启后，会自动从系统 U 盘启动，进入 Win11 安装界面，依次选择语言、时区、键盘布局，点击「现在安装」；

5. 到输入产品密钥页面，选择「我没有产品密钥」，后续进入系统再激活即可，不影响安装；

6. 到「你想将 Windows 安装在何处？」页面，此时列表里只有 U 盘，看不到本地硬盘，点击加载驱动程序选项，弹出驱动查找窗口，点击「浏览」，找到另一个驱动 U 盘，打开之前解压好的 Intel RST 驱动文件夹，选中名称包含 Intel RST VMD Managed Controller 的那一项，选中后点击确定；如果列表里只有一个 RST 相关驱动，直接选它即可，不会选错；

7. 等待驱动加载完成，大概 5-10 秒，加载过程中不要拔 U 盘、不要关机；

8. 驱动加载完成后，回到安装位置页面，就能看到电脑自带的 NVMe 固态硬盘正常显示在列表里，选中识别到的本地硬盘；

9. 继续下一步，此时需要链接网络，推荐跳过联网，创建本地账户，后续进桌面再联网登录微软账号即可，可以避免系统自动把用户文件夹名称设为邮箱前 5 位；

10. 根据机型选择按 <kbd>Fn</kbd> + <kbd>Shift</kbd> + <kbd>F10</kbd> 或 <kbd>Shift</kbd> + <kbd>F10</kbd> 调出命令窗口，输入命令 `oobe\bypassnro`，按下回车，电脑会自动重启。重启后回到设置向导，选择我没有 Internet 连接 → 继续执行受限设置，此时就能自定义用户名（建议纯英文和数字组合，避免软件路径报错），顺利进入桌面后，再连接网络，后续想绑定微软账户，在设置里登录即可，用户文件夹名称保持你自定义的样子，不会乱码。

# 删除双系统残留

## 删除开机双系统引导项

因为之前有旧 Win10 系统残留，开机每次都会弹出系统选择界面，删除引导项即可：

1. 按下 <kbd>Win</kbd> + <kbd>R</kbd>，输入 `msconfig` 回车，打开系统配置；

2. 切换到引导标签页，选中旧 Win10 启动项，点击删除，确定后重启，开机直接进 Win11。
